# OpenCode Model Comparison Report

Date: 2026-04-26

Scope: 对 OpenCode 平台上的六个模型进行对比，聚焦工作适配性、能力边界与典型任务使用场景。

Note: OpenCode Zen 作为模型编排平台，负责在会话中切换不同模型、实现容错与回退策略；下列对比聚焦于六个具体模型本身。

## 模型列表
- Big Pickle
- GPT-5 Nano
- H3 preview Free
- Ling 2.6 Flash Free
- MiniMax M2.5 Free
- Nemotron 3 Super Free

## 快速要点（No-nonsense 版）
- Big Pickle：强推理与跨领域能力，适合复杂任务与长线推理；资源与成本相对较高。
- GPT-5 Nano：通用型强、速度快、成本相对友好；适合日常开发与快速迭代。
- H3 preview Free：轻量级、免费等级，适合简单任务与草拟，能力受限于复杂场景。
- Ling 2.6 Flash Free：中文语言能力突出，适合中文文档、中文文本处理。
- MiniMax M2.5 Free：极低成本、适合极简任务，但能力有限。
- Nemotron 3 Super Free：通用性平衡，适合日常文本处理/轻度数据分析。

## 工作场景与选型建议
- 需要高质量、复杂推理、长文本分析或架构设计：优先 Big Pickle 或 GPT-5 Nano。
- 需要快速迭代、日常开发辅助、脚本生成：GPT-5 Nano 最合适；必要时与 Big Pickle 组合以获得更可靠的推理。
- 以中文为核心的文本处理、文档撰写与润色：Ling 2.6 Flash Free 或 Nemotron 3 Super Free。
- 轻量任务、演示与原型：H3 preview Free、MiniMax M2.5 Free。
- 需要稳定的并发处理和容错机制：优先使用具备回退和熔断机制的模型组合，如多模型并行与自动回退策略的方案。

## 实操建议
- 将长线任务放在高阶模型上，短期、简单任务放在免费模型上对比输出。
- 对同一任务用不同模型并行执行，记录输出差异、响应时间与成本，形成对比表。
- 对预算有限的场景，建立轮换策略，确保关键任务有可用的高阶模型。
- 使用统一评审标准对输出进行一致性检查，确保可追溯。
